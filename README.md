# on-ayx-journal

## 项目简介

`on-ayx-journal` 是一个用于归档和发布独立 HTML 页面的仓库。  
这里存放的页面彼此独立，不绑定特定的域名或网站结构，适合作为静态内容集合的存储与展示空间。

## 目录说明

```
.
├── index.html          # 入口页面，可浏览归档列表
├── pages/              # 存放独立 HTML 页面
├── assets/             # 公共资源（样式、脚本、图片等）
├── README.md           # 本文件
└── LICENSE             # 开源许可（如有）
```

- `pages/` 目录下每个子目录或文件代表一个独立的归档页面。
- `assets/` 用于存放多个页面共享的静态资源。
- 目录结构可根据归档需求灵活调整。

## 页面归档说明

- 每个页面为完整的、可独立打开的 HTML 文件。
- 页面之间一般不互相引用，保持解耦。
- 归档内容以静态形式保存，不依赖后端服务或数据库。
- 如页面包含外部资源（字体、库等），建议在文档中注明来源。

## 维护说明

- 欢迎通过 Pull Request 提交新页面或修正现有内容。
- 提交时请确保页面可正常渲染，无损坏的资源引用。
- 如对归档结构有建议，可提交 Issue 讨论。
- 本仓库仅作为存档用途，不保证页面内容的持续更新或可用性。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任一 HTML 文件。
3. 也可部署到任意静态托管服务（如 GitHub Pages、Netlify 等）进行在线浏览。
